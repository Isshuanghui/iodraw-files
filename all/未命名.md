```mermaid
sequenceDiagram
    title 秒杀下单核心流程时序图
    actor 用户
    participant Browser
    participant Controller
    participant Service
    participant Redis
    participant MQ_PROD
    participant MQ_CONS
    participant MySQL
    participant ES

    %% 第一步：用户发起秒杀请求
    用户->>Browser: 提交秒杀请求
    Browser->>+Controller: POST /doSeckillProduct
    Controller->>+Service: doSeckillProduct(entity)

    %% Service 前置校验流程 1~5
    Service->>+Redis: 1.校验Token+每日限流
    Redis-->>-Service: Token有效/未超限
    Service->>+Redis: 2.验证图片验证码
    Redis-->>-Service: 验证码正确
    Service->>+Redis: 3.检查秒杀时间窗口
    Redis-->>-Service: 在有效范围内
    Service->>+Redis: 4.检查售罄标识
    Redis-->>-Service: 未售罄
    Service->>+Redis: 5.原子预扣减 DECR stock
    Redis-->>-Service: stock=15 (大于0)

    %% alt 互斥分支：库存不足 / 重复秒杀 / 秒杀成功
    alt [库存不足]
        Service->>+Redis: 设置售罄标识(3600s)
        Service-->>-Controller: 抛出BusinessException
        Controller-->>Browser: 已售罄
    else [重复秒杀]
        Service->>+Redis: 检查用户秒杀记录
        Redis-->>-Service: 记录已存在
        Service-->>-Controller: 抛出BusinessException
    else [秒杀成功]
        Service->>+Redis: 6.INCR每日计数
        Service->>+Redis: 7.设置状态 0:10排队中
        Service->>+MQ_PROD: 8.asyncSend(秒杀消息)
        MQ_PROD-->>-Service: 发送完成
        Service-->>-Controller: ApiResult[排队中]
        Controller-->>Browser: 显示进度条(10%)
    end

    %% MQ消费者异步处理下单逻辑（MQ_CONS 消费消息）
    MQ_PROD->>+MQ_CONS: 推送秒杀消息
    MQ_CONS->>+Service: createOrder(entity)
    Service->>+Redis: 9.更新状态 0:50:处理中
    Redis-->>-Service: 更新成功
    Service->>+MySQL: 10.UPDATE seckill_product SET qty=qty-1 WHERE id=? AND qty>0
    MySQL-->>-Service: 扣减成功
    Service->>+Redis: 11.同步库存 SET
    Redis-->>-Service: 同步完成
    Service->>+Redis: 12.更新状态 0:75:创建订单
    Redis-->>-Service: 更新完成
    Service->>+MySQL: 13.INSERT INTO seckill_order_trade
    MySQL-->>-Service: orderId=17892345
    Service->>+ES: 14.更新ES商品库存
    ES-->>-Service: 更新完成
    Service->>+Redis: 15.存储成功记录
    Redis-->>-Service: 存储完成
    Service->>+Redis: 16.状态 1:100:成功
    Redis-->>-Service: 更新完成
    Service->>+MQ_PROD: 17.发送超时延迟消息(15min)
    MQ_PROD-->>-Service: 发送完成
    deactivate Service
    deactivate MQ_CONS

    %% 前端轮询查询订单结果
    Browser->>+Controller: GET /getResult?productId=xxx
    Controller->>+Service: getResultWithProgress()
    Service->>+Redis: 读取处理状态
    Redis-->>-Service: 1:100:订单创建成功
    Service-->>-Controller: SeckillResultResponse.success()
    Controller-->>-Browser: 返回成功数据
    Browser->>用户: 展示成功页面 + 订单详情
```
